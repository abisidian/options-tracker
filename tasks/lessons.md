# Lessons

- 策略名称如果同时涉及方向、期权类型和开仓方式，界面上不要只显示 `BPS`、`BCS` 这类歧义缩写；优先显示能直接表达“买/卖价差”属性的文案，避免用户把 `Bull Put` 误读成 `Bear Put`
- 用户已经明确要求策略标签使用英文术语；涉及期权策略名称时，不要擅自改成中文，应优先使用像 `Sell Call Spread`、`Sell Put Spread` 这种直接表达开仓方式的英文文案
- 调整策略文案时，要同时检查表格、切换 tab、图表标题等所有用户可见入口，避免同一策略在不同组件中出现两套命名
- 线上“没生效”先核对远端 `HEAD` 是否包含目标改动；如果本地文件仍是未提交状态，不要先把问题归因到时序或缓存
